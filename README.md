## 🌿 Problème
 
Les entreprises déploient des modèles d'IA de plus en plus lourds — mais **personne ne mesure vraiment leur coût environnemental**.
 
- 🔥 L'entraînement d'un LLM peut émettre autant de CO₂ qu'un vol transatlantique
- ⚡ L'inférence quotidienne de milliers de requêtes = consommation électrique massive et invisible
- 🏭 Des datacenters sous-optimisés = gaspillage caché qui explose les coûts
 
**SPARK IA** rend cet impact visible, mesurable et actionnable.
 
---
 
## ✨ La solution
 
Plateforme SaaS conçue pour les **grandes entreprises** qui veulent piloter l'empreinte carbone de leurs systèmes d'IA.
 
| Module | Ce que ça fait |
|--------|---------------|
| 📊 **Dashboard** | Vue temps réel de la consommation énergétique et des émissions CO₂ par projet IA |
| 🔌 **Sources de données** | Connexion à AWS, Azure, GCP, OpenAI API, Anthropic API, Datadog, SAP... |
| 🧪 **Simulateur** | Anticiper l'impact d'un changement de modèle ou de datacenter avant de migrer |
| 🏗️ **Infrastructure** | Cartographie des datacenters avec PUE, intensité carbone, conformité RGPD |
| 📋 **Rapport RSE** | Génération automatique du rapport d'impact IA · format CSRD / ESG ready |

## 🎯 Fonctionnalités interactives
 
- **Navigation complète** entre les 5 modules (Dashboard, Sources, Simulateur, Infrastructure, RSE)
- **Simulateur dynamique** : changez le modèle IA, le volume de requêtes, le datacenter → les économies CO₂ et financières se calculent en temps réel
- **Connexion/déconnexion** des sources de données simulées (AWS, Azure, GCP, SAP...)
- **Alertes et recommandations** générées automatiquement par l'IA
- **Score "IA Responsable"** calculé pour chaque projet

## 🛠️ Stack technique
 
```
Frontend  →  HTML5 · CSS3 (custom properties, animations) · JavaScript vanilla
Fonts     →  Cabinet Grotesk · Instrument Mono (Google Fonts)
Design    →  Interface dark mode, thème SPARK IA, composants temps réel
Data      →  Simulation de métriques réalistes (CO₂, kWh, coûts cloud)
```
 
Aucune dépendance externe, aucun framework — 100% fichier unique.

## 📊 Exemple de résultats du simulateur
 
> Pour un projet de Détection de fraude avec 22 000 requêtes/jour sur GPT-4 (US-East) :
 
| Scénario | CO₂/an | Coût API/an | Énergie/an |
|----------|--------|-------------|-----------|
| **Actuel** (GPT-4 · US-East) | 5.9 t CO₂ | ~128 k€ | 18.6 GWh |
| **Optimisé** (Mistral 7B · EU-West) | 0.05 t CO₂ | ~12 k€ | 274 MWh |
| **Réduction** | **−99%** | **−116 k€/an** | **−98%** |
